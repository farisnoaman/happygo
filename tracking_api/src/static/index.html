API Endpoints

POST /api/location

Update a single driver location

{ "driver_id": "driver123", "latitude": 37.7749, "longitude": -122.4194, "speed": 25.5, "heading": 180.0, "accuracy": 5.0, "is_offline": false, "trip_id": "TRIP-2025-001", "timestamp": "2025-07-26T15:30:00Z" }
POST /api/location/batch

Update multiple driver locations in a single request

{ "locations": [ { "driver_id": "driver123", "latitude": 37.7749, "longitude": -122.4194, "speed": 25.5, "timestamp": "2025-07-26T15:30:00Z" }, { "driver_id": "driver123", "latitude": 37.7750, "longitude": -122.4195, "speed": 26.0, "timestamp": "2025-07-26T15:30:30Z" } ] }
GET /api/location/{driver_id}

Get location history for a specific driver

Query Parameters:

  • hours - Number of hours to look back (default: 24)
  • limit - Maximum number of records (default: 1000)
GET /api/location/{driver_id}/latest

Get the most recent location for a specific driver

POST /api/sync/{driver_id}

Manually trigger synchronization to Frappe for a specific driver

GET /api/sync/status

Get synchronization status for all drivers

GET /api/health

Health check endpoint for monitoring

Configuration

Set the following environment variables for Frappe integration:

FRAPPE_BASE_URL=http://your-frappe-site.com FRAPPE_API_KEY=your_api_key FRAPPE_API_SECRET=your_api_secret

Response Format

All endpoints return JSON responses with the following structure:

Success Response:
{ "status": "success", "message": "Operation completed successfully", "data": { ... } }
Error Response:
{ "status": "error", "message": "Error description" }

Usage Examples

Update Location (cURL)

curl -X POST http://localhost:5000/api/location \ -H "Content-Type: application/json" \ -d '{ "driver_id": "driver123", "latitude": 37.7749, "longitude": -122.4194, "speed": 25.5 }'

Get Driver Locations (cURL)

curl "http://localhost:5000/api/location/driver123?hours=1&limit=100"
